# Spring-Basic-Week1

## 프로젝트란?

- 하나의 목표를 세우고 달성하기 위한 모든 과정들
- 목표 달성을 위해 계획, 개발, 관리되는 일련의 작업

## 프레임워크 vs 라이브러리

- **프레임워크(Framework)**: 소프트웨어 개발에서 애플리케이션의 기본 구조를 제공하는 클래스와 라이브러리의 집합, 흐름을 프레임워크가 제어
- **라이브러리(Library)**: 자주 사용되는 코드, 함수, 클래스, 자료형 등을 미리 작성하여 모아둔 재사용 가능한 자원의 집합, 개발자가 필요할 때 호출

## 스프링 부트의 등장

- **스프링 프레임워크(Spring Framework)**: 자바 플랫폼을 위한 오픈 소스 애플리케이션 프레임워크, 기능이 많지만 설정이 복잡함

스프링 프레임워크는 많은 기능을 제공하는 대신 설정이 어려웠고, 웹/서비스 개발 수요가 증가하며 빠른 개발이 중요해짐 -> 스프링을 쉽고 빠르게 쓰기 위한 자동화 도구로 스프링부트가 등장

- **스프링 부트(Spring Boot)**
  - 자바 스프링 프레임워크 기반의 애플리케이션을 복잡한 설정 없이 쉽고 빠르게 개발할 수 있게 해주는 도구
  - 설정 자동화, 내장 서버, 빠른 실행 환경 제공
