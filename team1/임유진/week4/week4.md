# 스프링의 주요 원리

## 1. IoC

- Inversion of Control, 제어의 역전
- **기존 방식**: 개발자가 직접 `new` 연산자로 객체를 생성하고 소멸시키며 흐름을 제어
- **역전된 방식**: 객체의 생성부터 생명주기(Lifecycle) 관리까지의 모든 제어권을 스프링 프레임워크가 가져감
- 개발자는 비즈니스 로직에만 집중할 수 있고, 객체 간 결합도가 낮아짐

## 2. 컨테이너

- IoC/DI 컨테이너
- 스프링에서 객체(Bean)를 생성하고, 서로 연결해 주며 관리하는 공간
- 애플리케이션이 실행될 때 필요한 객체들을 미리 만들어 이 컨테이너에 담아둠
- 최근에는 객체 간의 관계를 맺어주는 역할이 강조되어 DI 컨테이너라고도 함

## 3. 스프링빈

- 스프링 컨테이너가 생성하고 관리하는 **자바 객체**
- 일반적인 자바 객체와 달리 컨테이너에 의해 라이프사이클이 제어됨

## 4. DI

- Dependency Injection, 의존성 주입
- 의존성 = 사용할 것 = 객체 (cf. 객체를 사용한다 = 객체를 생성해서, 메소드/필드를 호출, 접근한다)
- 스프링에게 **필요한 객체를 요청하여 주입** 받는 방식으로 DI가 이루어짐
